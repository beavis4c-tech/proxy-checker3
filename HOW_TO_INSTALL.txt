================================================================================
  HOW TO INSTALL — Cloud Proxy Checker on Render (Web Service)
================================================================================

Prerequisites:
  - A GitHub account (free)
  - A Render account (free tier works) at https://render.com
  - Your Discord webhook URL ready

================================================================================
  STEP 1 — Push the cloud_checker/ folder to GitHub
================================================================================

  1. Create a new GitHub repository (public or private).
  2. Upload the entire "cloud_checker" folder to the root of that repo.
     Your repo should look like:

       your-repo/
       └── cloud_checker/
           ├── cloud_proxy_checker.py
           ├── webhook.txt
           ├── requirements_cloud.txt
           ├── render.yaml
           └── start.sh

  3. Make sure webhook.txt contains your Discord webhook URL (one line, no
     extra spaces). You can edit it directly on GitHub after pushing.

================================================================================
  STEP 2 — Create a new Web Service on Render
================================================================================

  1. Log in to https://dashboard.render.com
  2. Click the "New +" button (top right) → select "Web Service".
  3. Connect your GitHub account if you haven't already.
  4. Select the repository you just pushed.
  5. Render will auto-detect the render.yaml file. If it doesn't, configure
     manually:

       Name:          proxy-checker (or any name you like)
       Region:        Oregon (us-west)
       Branch:        main
       Runtime:       Python 3
       Build Command: pip install -r cloud_checker/requirements_cloud.txt
       Start Command: python cloud_checker/cloud_proxy_checker.py
       Plan:          Free

  6. Scroll down and click "Create Web Service".

================================================================================
  STEP 3 — Verify it's running
================================================================================

  1. Render will build and deploy. This takes 1-2 minutes.
  2. Once deployed, open the "Logs" tab in your Render dashboard.
  3. You should see:

       =======================================================
         CLOUD PROXY CHECKER -- Infinite Loop
         Target : http://httpheader.net/
         Threads: 500  |  Timeout: 12s  |  Batch: 10000
         Webhook: https://discord.com/api/webhooks/...
       =======================================================

  4. The checker will now run forever — generating 10,000 proxies per batch,
     checking them with 500 threads, and sending every HIT to your Discord.

================================================================================
  TROUBLESHOOTING
================================================================================

  Q: "webhook.txt not found" error in logs?
  A: Make sure webhook.txt is inside the cloud_checker/ folder and pushed
     to GitHub. The file must be named exactly "webhook.txt".

  Q: Web service keeps sleeping?
  A: Free-tier Render services sleep after 15 minutes of inactivity. Since
     this script runs an infinite loop, it should stay awake. If it sleeps,
     upgrade to a paid plan or use a cron-job pinger (e.g., cron-job.org)
     to hit your service URL every 10 minutes.

  Q: Want to change the webhook without redeploying?
  A: Edit webhook.txt directly on GitHub. Render will auto-redeploy on push.
     Or use Render's "Shell" tab to edit the file live (changes will be lost
     on next deploy unless you also update the repo).

  Q: How do I stop it?
  A: In the Render dashboard, click your service → "Suspend Web Service".
